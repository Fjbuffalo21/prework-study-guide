Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The title element defines the title for the webpage.
  • The style element defines the css styles associated with a website.
  • The base element defines the url for a webpage.
  • The link element connects an external resource to the HTML document
  • The meta element defines metadata such as the character set, description, keywords, author, and viewport.
  • The script element embeds client-side scripts in the html document

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside the element.
  • Text-align is one way to center a title or heading on a webpage.
  • Img selector assigns property value to any image element in the markup
  • The display property is assigned with the block value which assigns each image to start on a new line.
  • The ref attribute specifies relationship between the current document and the linked document resource.
  • The href attribute specifies te location of the URL of the external resource.
  • The class atrribute allows us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector.
  • The card attrbute designates this as a class selector.
  • The width attribute assigns the width.
  • The border attribute assigns the border.
  • THe box-shadow attribute adds shadow effects around the elements frame.

Git

  • git status: checks what branch we are currently working on.
  • git checkout -b branch-name: creates a new branch and swithces to it.
  • git add [file]: add a file as it looks now to your next commit.
  • git reset [file]: unstage a file while retaining those changes in the working directory.
  • git diff: takes two input data sets and outputs the changes between them.
  • git diff--staged: only show changes to files in the "staged" area.
  • git commit-m "[descriptive message]": commit your staged content as a new commit snapshot.
